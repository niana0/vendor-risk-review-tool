# Vendor Security Assessment Tool

A Streamlit web app for GRC vendor risk reviews at Datadog. Works in two phases: an initial review generated by Claude (via the `/vendor-risk-review` skill), followed by document analysis through one of three review modes.

---

## Features

- **Three review modes** via a tabbed UI — each is self-contained with its own inputs and outputs
- **Vendor Documentation Review** — parse SOC 2, pen test reports, and policies; generate a Final Vendor Risk Review when an initial review exists
- **Completed Questionnaire Review** — read a vendor-answered questionnaire and flag security findings (High / Medium / Low)
- **Auto-Fill Questionnaire** — use RAG to answer your blank security questionnaire from vendor docs
- **Dark mode** UI by default, with light/dark toggle
- **Vendor-named outputs** — all files prefixed with the vendor slug

---

## How It Works

### Two-Phase Workflow

**Phase 1 — Initial Review (Claude)**
Run `/vendor-risk-review <ticket-id>` in Claude Code. Claude fetches Jira tickets, searches the web, applies triage criteria, and saves a structured review to `output/{vendor_slug}_review.md`. The Streamlit app picks this up automatically.

**Phase 2 — Document Analysis (Streamlit)**
Upload vendor documents to one of three tabs:

| Tab | Input | What it does |
|---|---|---|
| Vendor Documentation Review | Vendor PDFs/Excel (multi) | Extracts structured findings from SOC 2, pen test, policies; generates Final Vendor Risk Review (if initial review exists) |
| Completed Questionnaire Review | Vendor-completed Excel | Reads vendor answers, flags security findings by severity, outputs annotated Excel |
| Auto-Fill Questionnaire | Vendor docs + blank Excel | Uses RAG + GPT-4o to answer each question from vendor docs, outputs completed Excel |

### Confidence Levels (Auto-Fill Questionnaire)
- **HIGH** — Question directly answered in the documentation
- **MEDIUM** — Partially addressed
- **LOW** — Only indirectly mentioned
- **NOT_FOUND** — Not in any uploaded document

### Finding Severity (Completed Questionnaire Review)
- **High** — Missing critical controls: no encryption, no DR/BCP, no MFA, no security policy, no pen testing
- **Medium** — Partial or informal controls, undocumented manual processes
- **Low** — Minor gaps, best-practice improvements

## Quick Start

### Prerequisites
- Python 3.10+
- OpenAI API key

### Setup

```bash
cd "[Directory where you saved the project]"
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

Create a `.env` file:
```
OPENAI_API_KEY=your_openai_api_key_here
```

### Run

```bash
streamlit run app.py
```

Opens at **http://localhost:8501**

---

## Using the App

1. Run `/vendor-risk-review <ticket-id>` in Claude Code (optional but recommended — enables Final Vendor Risk Review)
2. Open the app — the initial review appears automatically if it exists
3. Choose a tab based on what you have:
   - **Vendor Documentation Review** — upload SOC 2, pen test, policy docs
   - **Completed Questionnaire Review** — upload a vendor-answered Excel
   - **Auto-Fill Questionnaire** — upload vendor docs + your blank questionnaire Excel
4. Click the process button and wait
5. Download the output report(s)
6. Click **Start New Review** to reset for the next vendor

---

## Output Files

All files are saved to `output/` prefixed with the vendor slug.

| File | Mode | Contents |
|---|---|---|
| `{vendor}_vendor_risk_summary.docx` | Vendor Documentation Review | Final Vendor Risk Review (Word) |
| `{vendor}_security_doc_review.docx` | Vendor Documentation Review | SOC 2, pen test, policy findings (Word) |
| `{vendor}_final_vendor_risk_review.md` | Vendor Documentation Review | Final review (Markdown) |
| `{vendor}_questionnaire_findings.xlsx` | Completed Questionnaire Review | Original Q&A + Finding column |
| `{vendor}_completed_questionnaire.xlsx` | Auto-Fill Questionnaire | Answered questionnaire with confidence + sources |

## Troubleshooting

**"OpenAI API key not configured"**
- Check `.env` exists in the project root with `OPENAI_API_KEY=sk-...`
- Restart the app after editing `.env`

**No questions found in questionnaire**
- Ensure the Excel has questions longer than 20 characters
- Check the first sheet has question and answer columns

**Port already in use**
```bash
lsof -ti:8501 | xargs kill -9
streamlit run app.py
```

**Module not found**
```bash
source venv/bin/activate
pip install -r requirements.txt
```
